Display Types¶

Typhon has two major widgets that users are expected to interface with. The first is the TyphonDisplay. This is the barebones implementation. No signals, or widgets are automatically populated in the screen. In fact, by default most of the widgets will be hidden. You can then manually add signals to the panels and plots, the panels will only show themselves when you add PVs.

For a more automated approach, the DeviceDisplay will take your ophyd Device and populate PVs and components into the appropriate locations

TyphonDisplay¶

class typhon.TyphonDisplay(name, image=None, parent=None)[source]¶

Generalized Typhon display

This widget lays out all of the architecture for a general Typhon display. The structure matches an ophyd Device, but for this specific instantation, one is not required to be given. There are four main panels available; read_panel, config_panel, method_panel. These each provide a quick way to organize signals and methods by their importance to an operator. Because each panel can be hidden interactively, the screen works as both an expert and novice entry point for users. By default, widgets are hidden until contents are added. For instance, if you do not add any methods to the main panel it will not be visible.

This device is the bare bones implementation in the event that someone might want to collect a random group of signals and devices together to create a screen. A more automated display generator is available in the DeviceDisplay

Parameters:
  • name (str) – Title displayed on the widget
  • image (str, optional) – Path to an image file to include in the display.

add_subdisplay(name, display, list_widget)[source]¶

Add a widget to one of the button layouts

This add a display for a subcomponent and a QPushButton that will bring the display to the foreground. Users can either specify their button or have one generate for them. Either way the button is connected to the pyqSlot show_subdevice()

Parameters:
  • name (str) – Name to place on QPushButton
  • display (QWidget) – QWidget to associate with button
  • button (QWidget, optional) – QWidget with the PyQtSignal clicked. If None, is given a QPushButton is created

DeviceDisplay¶

class typhon.DeviceDisplay(device, methods=None, image=None, parent=None)[source]¶

Bases: typhon.display.TyphonDisplay

Display an Ophyd Device

Using the panels created by the inherited TyphonDisplay the sub-devices and signals are added to the appropriate places in the widget. The display introspects the read_attrs, configuration_attrs, component_names and _sub_devices to find the signal names and heirarchy.

Parameters:
  • device (ophyd.Device) – Main ophyd device to display
  • methods (list of callables, optional) – List of callables to pass to FunctionPanel.add_method()
  • image (str, optional) – Path to image to add to display
